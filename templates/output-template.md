
- 价格热力色：最低价绿色`¥{价}`、中等黄色`¥{价}`、最高价红色`¥{价}`

### B2. 写报告的四个硬性要求

| 要求 | 具体做法 | 反例 |
|------|---------|------|
| **必写短板** | 每个推荐后面跟上 ⚠️ "这个方案不能做什么" | 只吹优点不提限制 |
| **直接给结论** | 用"买 XXX ¥XX/月"这种指令式语言 | "可以考虑""也可以选择"这种模糊话术 |
| **敢说不推荐** | 把性价比极低的套餐单独列出来，标注"垃圾套餐，别碰" | 汇总所有方案让用户自己挑 |

## 通用规则（两赛道共用的交付规范）

### 交付方式
| 方式 | 内容 |
|------|------|
| 聊天窗口 | 贴核心结论 + 对比卡 + 推荐 + 最新变化 |
| MD 文件 | 完整报告（带赛道标记 + 时间戳文件名） |

### 文件命名
- API中转站：`API中转站_模型名_YYYY-MM-DD.md` 或 `API_Proxy_Compare_YYYYMMDD.md`
- Token Plan：`TokenPlan_采购决策_YYYY-MM-DD.md` 或 `Token_Plan_Analysis_YYYYMMDD.md`
- 禁止混用命名，禁止无时间戳

### 赛道标记强制要求
- 所有输出（聊天 + 文件）第一行标注赛道
- API中转站：`🔄 API中转站 | 按量付费`
- Token Plan：`📦 Token Plan | 包月订阅`
