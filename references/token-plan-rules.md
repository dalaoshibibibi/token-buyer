# Token Plan / Coding Plan 对比逻辑 v2

## 一、三大计价逻辑分类（必须对照分类选择算账方法）

### 第1类：固定配额型（真·Coding Plan）
月费 → 固定 token/调用次数 → 用完即停/等刷新
- **代表**：MiniMax / GLM / Kimi
- **核心算账**：折扣率（元/万token vs API 按量价）
- **窗口限制**：5h+周窗口严格

### 第2类：模型超市型（Credits 兑换制）
月费 → 通用 Credits → 各模型按不同速率消耗
- **代表**：Qwen / 火山引擎方舟 / 腾讯 TP
- **核心算账**：综合折扣率 + 模型覆盖度 + 旗舰模型抵扣系数
- **注意**：折扣率通常 > 100%，卖的是"超市入场券"

### 第3类：API 资源站型（预付费 Credits + 模型折扣）
月费 → USD/CNY Credits → 按模型级别定价消耗
- **代表**：Command Code / Open Code Go
- **核心算账**：模型级有效花费（按常用模型算有效元/次）
- **注意**：与标准 API 中转站同赛道对比，不适用传统折扣率计算

### 双赛道并行原则
- **赛道一**：固定配额型 + 模型超市型 → 采用折扣率 + 模型灵活性 2 维
- **赛道二**：API 资源站型 → 采用模型级有效花费，与标准 API 中转站（Zenmux/Haoshuang/Yunwu）同锚点对比
- **最终汇合**：两个赛道在"按场景推荐"表里统一输出，但不跨赛道算折扣率

## 四、Token Plan 对比输出规范（v2 — 双赛道）

### 赛道分流原则
1. 先判断目标 Coding Plan 属于哪个分类：
   - **赛道一（固定配额型+模型超市型）**：MiniMax/GLM/Kimi/Qwen/火山引擎方舟/腾讯TP → 三段式对比
   - **赛道二（API 资源站型）**：Command Code / Open Code Go → 模型级有效花费对比，**不与赛道一混比折扣率**
2. 两条赛道产生独立报告，在"按场景推荐"最终输出时汇合
3. 跨赛道组合方案必须标注"跨赛道组合"

### MD报告固定章节（v4.0 三段式）
**第一部分：分析过程**
1. ⭐ **真实折扣反推表**（v4.0 核心）— 每个套餐的等效 API 价值、实际折扣率
2. 五平台套餐总览表（价格/额度/超额方式）
3. 核心维度对比（Token额度/调用次数/窗口限制/多模态能力/模型覆盖）
4. 关键发现：每个维度一句话总结

**第二部分：决策推荐**
1. **使用画像反推** — 如未明确场景，先问月均调用/主要场景/团队规模
2. **按预算分档推荐**（≤100 / 100-200 / 200-600 / 600-1500）
3. **组合套餐方案** — 列出 2-3 个组合 vs 单套餐对比
4. 每个推荐必须含 ✅ 理由 + ⚠️ 短板

**第三部分：速查卡片**
1. 决策速查表（预算→买谁→一句话理由）
2. 刚需对照表（如果要 X → 唯一选择是 Y）
3. 避坑清单（直接说哪些别碰 + 原因）
4. 真实折扣率排名表
5. 数据来源与备注

### 对比表必须包含的列（分平台逐项列出）
- 套餐名称 / 月费 / 年付优惠
- 月 token 量 / 月调用次数（估算）
- 5小时窗口额度 / 周窗口额度
- Agent 并发数
- 模型范围 / 多模态支持 / 视频能力
- **每万token成本折算**（月费 ÷ 月总token量 × 10000）
- **等效API价值**（折合真实的 API 调用需要花多少钱）
- **实际折扣率**（月费 ÷ 等效API价值 × 100%，<5% = 极高性价比）

### 真实折扣反推规则（v4.0 新增，不可跳过）
1. 读取 `references/api-base-price.md` 获取各平台旗舰模型的真实 API 单价
2. 按混合均价（输入+输出÷2）计算"等效 API 价值"
3. 实际折扣率 = 套餐月费 ÷ 等效 API 价值 × 100%
4. 折扣率 > 100% → 标注"该套餐卖的不是 token 折扣，是附加价值"
5. 报告必须包含"真实折扣排名表"（从低到高排列）

## 五、抓取指南
1. **优先找官方说明书**：/docs/ 路径下通常有详细限额说明
2. **WebFetch入口页面**：获取套餐价格、能力列表等展示信息
3. **翻说明书文档**：获取 5小时/每周 窗口的精确数字
4. **注意活动时效**：限时扩容、限时折扣等需要标注有效期
5. **第三方对比站**：如 codingplan.org 可作参考，需回官方验证
