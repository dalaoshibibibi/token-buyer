# Token买手 版本演进记录

> 此文件记录历史变更，供 AI 执行时不需要加载。

### v6.0 — 2026-06-30：三大计价逻辑分类 + 双赛道
**问题**：Command Code（Credits+Deals型）和 Open Code Go（预付费USD额度型）与传统 Token Plan 算账逻辑不同。

**改动**：
- 三大计价逻辑：固定配额型 / 模型超市型 / API资源站型
- Command Code / Open Code Go 归入"API 资源站型"，与 API 中转站同锚点对比
- 新增 Open Code Go 收录（$10/月，13款模型）
- 所有输出文件强制带时间戳

### v5.0 — 2026-06-29：两赛道分离 + 单模型对比卡
**问题**：API中转站（按量付费）和 Token Plan（包月订阅）混在一份报告，用户分不清。

**改动**：
- 入口分流：用户触发后先确认赛道，再执行
- 新增 API 中转站「单模型对比卡」交互
- 报告头部强制标注赛道

### v4.0 — 2026-06-28：真实折扣反推 + 使用画像 + 组合套餐
**问题**：v3.0 价格数据缺乏验证，推荐结论无法信任。

**改动**：
- 新增「真实折扣反推」：用真实 API 单价反算套餐等效价值 + 折扣率
- 新增「使用画像反推」：先问 3 个问题（调用量/场景/团队），再用答案推荐档位
- 新增「组合套餐视角」：2-3 个组合方案 vs 单套餐对比
- 关键认知：Kimi Moderato 折扣率 0.8% 全网最低；Qwen 折扣率 >100% 卖的是模型超市入场券

### v3.0 — 2026-06-27：三段式输出结构
**问题**：v2.0 输出全是数据表格，用户看完不知道买谁。

**改动**：
- 输出重构：「分析过程 + 按预算分档推荐 + 速查卡片」三段式
- 强制规则：每个推荐必写短板、避坑清单、预算分档、速查卡
- 关键认知：GLM Lite 5h 仅 80次 prompts → 垃圾套餐；MiniMax Max ¥119 是综合性价比之王

### v2.0 — 2026-06-27：Token Plan 分析能力 + 爬取层
- 新增能力B（Token Plan 对比），WebFetch 驱动数据采集
- 收录 MiniMax、GLM、Kimi 三大 Token Plan

### v1.0 — 2026-06-26：基础版
- 三大 API 中转站比价（GeekNow、GrsAI、APIMart）
- Excel 导入 + 价格标准化 + 跨平台对齐
